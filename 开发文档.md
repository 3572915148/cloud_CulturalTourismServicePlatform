基于springboot的景德镇文旅服务平台
1.核心功能：AI文旅推荐
功能描述：1.接收用户请求 → 2. 查询匹配资源 → 3. 构建提示词 → 4. 调用AI模型 → 5. 解析响应 → 6. 返回结果

2.用户端功能
2.1用户注册、登录
前端：提供注册/登录表单，进行基础校验。

注册： 接收前端数据，检查用户账号是否已存在，然后存入数据库。登录： 验证用户名和密码，成功后生成一个唯一令牌（Token，如JWT），返回给前端。

后续请求： 前端在每次请求API时在Header中携带此Token，后端进行验证以识别用户身份。

2.2个人信息管理
实现：提供前端页面让用户查看和修改个人信息（昵称、头像、简介等）。后端提供相应的CRUD的API接口。

2.3文旅信息查询
数据来源： 初期可由你手动录入或通过网络爬虫（注意法律风险）获取本地文旅资源数据（景点、酒店、餐厅、博物馆等），建立数据库。

后端API： 提供分页查询、条件筛选（按类型、区域、评分等）的接口。

前端展示： 以列表或者页面的形式展示信息查看详情。
2.4在线预定
流程： 用户选择产品（如酒店房间、旅游套餐） -> 填写预订信息（日期、人数） -> 生成订单（包含价格、规则） -> 用户确认 -> 支付 -> 预订成功。

支付集成： 模拟支付流程。即用户点击“支付”后，订单状态直接变为“已支付”。

2.5陶瓷文化
实现：前端页面展示文章、图片、视频，介绍陶瓷历史、工艺、名家等。后端管理内容，前端进行展示。可以设计为分类查询的形式。

2.6评价与反馈
评价： 用户在订单完成后可以对购买的产品进行评分和文字评价。后端提供提交评价和查询评价列表的API。

反馈： 提供一个表单，让用户提交对平台的意见和建议。


商户端功能
3.1商户注册与入驻
流程与用户注册类似，但需要提交更多资质信息。后端需要增加一个审核状态字段。审核通过后，商户才能正式入驻。

3.2店铺管理
商户登录后，可以编辑自己的店铺信息（名称、Logo、介绍、联系方式、地址等）。后端提供对应的API，确保商户只能修改自己的店铺信息。

3.3产品管理
提供产品的增、删、改、查功能。产品信息包括标题、描述、价格、库存、图片等。

3.4评价管理
 商户可以查看用户对自己产品的评价，并可以做出回复。这是一个简单的查看和回复功能

技术框架

后端：springboot + spring + spring AI + mybatis plus + mybatis + spring Security
前端： vue + node.js
